## Диагноз: переобучение после 25-й эпохи

Модель достигает пика val_acc на 24-й эпохе, затем быстро деградирует:

```
Epoch 24: val_acc=0.139  train_loss=5.13
Epoch 28: val_acc=0.118  train_loss=4.79  (4 эпохи — потеря 0.021)
```

**Причина:** 423K параметров на ~1000 обучающих сэмплах (15 клипов).
Модель начинает запоминать train-специфичные визуальные паттерны,
которые не переносятся на val.

## Фундаментальное ограничение

Потолок val_acc ≈ 0.14 определяется объёмом данных (15 клипов).
Все архитектурные и регуляризационные изменения позволили сдвинуть
этот потолок с 0.115 до 0.139, но кардинальный рост возможен только
с расширением датасета.
