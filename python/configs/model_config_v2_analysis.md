# Анализ конфигурации model_config_v2

## Проблема

Модель переобучается на маленьком датасете (15 клипов). Конфигурация v1 избыточна по ёмкости.

## Изменения и обоснование

| Параметр | v1 | v2 | Причина |
|---|---|---|---|
| `d_model` | 128 | 64 | Вдвое меньше представление — меньше ёмкость |
| `nhead` | 8 | 4 | Пропорционально d_model (head_dim=16) |
| `num_layers` | 4 | 2 | Меньший датасет → меньше слоёв |
| `dim_feedforward` | 256 | 128 | 2×d_model — стандартное соотношение |
| `dropout` | 0.2 | 0.4 | Главный инструмент против переобучения |
| Conv3d out | 64 | 32 | Меньше карт признаков |
| backbone | 64→128→256 | 32→64 | Убран второй слой, уменьшены каналы |

## Стратегия

1. **Меньше каналов в CNN** — уменьшает число параметров в визуальном фронтенде
2. **Убрать один слой backbone** — меньше глубина, меньше шанс запомнить обучающие примеры
3. **d_model и dim_feedforward вниз** — снижает ёмкость трансформера
4. **num_layers: 4 → 2** — меньше слоёв внимания при маленьком датасете
5. **dropout: 0.2 → 0.4** — усиленная регуляризация, главный инструмент борьбы с переобучением

## Дополнительно

Убран padding `[2,2,2]` у Conv3d (заменён на `[1,1,1]`). Padding=2 при kernel=3 давал нечётное выходное время T, что вынуждало делать interpolation маски паддинга в encoder. С padding=1 при kernel=3 и stride=1 по временному измерению T на входе равно T на выходе.

## Использование

В `train.ipynb` заменить строку загрузки конфига:

```python
MODEL_CONFIG = load_config("configs/model_config_v2.json")
```
