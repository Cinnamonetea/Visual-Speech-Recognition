## Контекст

Наблюдаемая картина обучения (конфиг `model_config_v5.json`, BPE-датасет):

| Эпоха | train_loss | val_loss | Состояние |
|---|---|---|---|
| 1 | 7.11 | 7.01 | — |
| 52 | 5.51 | **5.97** | лучшая модель |
| 100 | 4.52 | 6.24 | переобучение |

Разрыв train/val loss вырос с 0.46 до 1.72 — классическое переобучение.
Причина: 392K параметров на 996 обучающих сэмплах (~400 параметров/сэмпл).

## Структурные проблемы текущего конфига

### Проблема 1: head_dim слишком мал

```
head_dim = d_model / nhead = 64 / 8 = 8
```

Стандартная рекомендация — `head_dim ≥ 32` (в оригинальном Attention is All You Need
использовалось 64). При `head_dim = 8` каждая голова attention работает в 8-мерном
подпространстве — слишком мало для моделирования осмысленных зависимостей между
кадрами или токенами.

### Проблема 2: dim_feedforward меньше d_model

```
dim_feedforward = 32 < d_model = 64
```

Стандартно: `dim_feedforward = 4 × d_model`. В текущем конфиге FFN в **два раза
меньше** самого d_model — каждый слой трансформера почти не добавляет
нелинейной выразительности поверх attention.

## Анализ числа слоёв (num_layers)

| Вариант | Параметры | За | Против |
|---|---|---|---|
| 4 слоя (текущий) | 392K | больше глубины | переобучается; слои слабые из-за `ffn=32` |
| 2 слоя | ~230K | меньше переобучение | меньше глубины для временных зависимостей |

**Вывод:** уменьшить `num_layers` 4→2 имеет смысл, но это не главное узкое место.
Качество каждого слоя (head_dim и dim_feedforward) важнее их количества.

## Применённые меры против переобучения

Внесены в `train_bpe.ipynb`:

| Мера | Было | Стало |
|---|---|---|
| Early stopping | нет | patience=25 эпох |
| Аугментация кадров | нет | яркость ×[0.7,1.3], контраст ×[0.8,1.2], шум σ=0.02 |
| dropout | 0.4 | 0.5 |
| learning rate | 3e-4 | 1.5e-4 |
| weight_decay | 1e-5 | 1e-3 |

Горизонтальный флип кадров **не применяется**: губы асимметричны при чтении.

## Рекомендуемый конфиг для следующего эксперимента (v6)

```json
{
  "d_model": 64,
  "nhead": 4,            // было 8  →  head_dim: 8 → 16
  "num_layers": 2,       // было 4
  "dim_feedforward": 128, // было 32  →  2×d_model вместо 0.5×d_model
  "dropout": 0.5,
  "vocab_size": 1200
}
```

При этих настройках:
- `head_dim = 64 / 4 = 16` — более разумный размер для attention
- `dim_feedforward = 128 = 2 × d_model` — FFN начинает реально работать
- Меньше параметров → меньше переобучение
- Каждый слой качественно сильнее текущих 4 слоёв со слабым FFN

## Фундаментальное ограничение

Все архитектурные и регуляризационные меры помогают, но не решают корневую проблему:
**15 клипов — крайне мало для lip reading**. Даже идеально настроенная модель
ограничена объёмом данных. Долгосрочный путь — расширение датасета или
предобучение на публичных корпусах (LRS2, LRS3, VoxCeleb).
