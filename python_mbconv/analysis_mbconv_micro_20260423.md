## Рекомендации по дальнейшему улучшению

### 1. Scheduled Sampling (заменить чистый Teacher Forcing)

**Проблема:** при teacher forcing decoder всегда получает правильные
предыдущие токены. Он учится предсказывать следующий токен в идеальных
условиях, которых не будет при инференсе (greedy decode без ground truth).
Это объясняет, почему val_acc=0.144 при teacher forcing, но инференс
выдаёт пустую строку.

**Решение:** постепенно заменять ground truth токены на предсказания
самой модели:

```python
# scheduled_sampling_ratio: 0→0.5 за 30 эпох
if random() < scheduled_sampling_ratio:
    prev_token = model_prediction  # собственное предсказание
else:
    prev_token = ground_truth      # teacher forcing
```

Это вынуждает decoder быть робастным к собственным ошибкам и
улучшает инференс без teacher forcing.

### 2. Заморозка encoder после пика

На эпохе 24 (пик val_acc) заморозить параметры encoder:

```python
if val_acc > best_val_acc:
    best_epoch = epoch
if epoch > best_epoch + 5:  # 5 эпох grace period
    for p in model.encoder.parameters():
        p.requires_grad = False
```

Это предотвращает запоминание encoder'ом train-специфичных паттернов
после достижения оптимальных визуальных представлений.

### 3. Ещё меньше параметров: d_model=32

Следующий логический шаг по тренду «меньше → лучше»:

```
d_model=32, nhead=2, ff=64
encoder=2, decoder=1
CNN: Stem(3→16) + MBConv(16→32) + MBConv(32→48)
Оценка: ~110K параметров
```

Но риск: модель может оказаться слишком маленькой для представления
задачи (как MBConv Light с агрессивной регуляризацией).

### 4. Расширение датасета (фундаментальное решение)

Все архитектурные эксперименты дали прирост от 0.127 до 0.144 (+13%).
Это потолок для 15 клипов. Добавление даже 10–20 клипов может дать
больший прирост, чем любые дальнейшие архитектурные изменения.
