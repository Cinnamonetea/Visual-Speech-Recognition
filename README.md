# Visual Speech Recognition

Модель визуального распознавания речи (lip reading) на основе архитектуры Transformer.
По видеозаписи области губ модель генерирует текстовую транскрипцию произнесённой фразы.

## Архитектура модели

Модель (`LipReadingTransformer`) реализована в `python/model.py` и состоит из трёх основных блоков:

### 1. Visual Encoder

Извлекает признаки из последовательности видеокадров и формирует контекстные векторы.

- **3D CNN Frontend** — обрабатывает пространственно-временные зависимости:
  - `Conv3d(3 -> 64)` с kernel `(5,5,5)` и stride `(1,2,2)` — захватывает движение губ по 5 соседним кадрам
  - `MaxPool3d` — дальнейшее пространственное сжатие
- **2D CNN Backbone** — извлекает пространственные признаки покадрово:
  - `Conv2d(64 -> 128 -> 256)` + `AdaptiveAvgPool2d` — сжимает каждый кадр в вектор размерности 256
- **Transformer Encoder** — моделирует временные зависимости между кадрами:
  - Позиционное кодирование (sinusoidal, max_len=40)
  - 4 слоя `TransformerEncoderLayer` (8 голов внимания, d_model=256)
  - Поддержка `src_key_padding_mask` для игнорирования паддинг-кадров

**Вход:** `[B, 40, 3, 96, 64]` — батч из 40 RGB-кадров размером 96x64  
**Выход:** `[B, 40, 256]` — контекстные векторы (memory)

### 2. Text Decoder

Авторегрессионный декодер, генерирующий токены текста по контексту из энкодера.

- `nn.Embedding(vocab_size, 256)` — эмбеддинги токенов
- Позиционное кодирование (sinusoidal, max_len=12)
- 4 слоя `TransformerDecoderLayer`:
  - Masked Self-Attention (causal mask) — модель видит только предыдущие токены
  - Cross-Attention — к выходу энкодера (memory)
  - Feed-Forward Network
- Линейный слой `fc_out` — проекция в пространство словаря

**Вход:** `[B, seq_len]` токены + memory от энкодера  
**Выход:** `[B, seq_len, vocab_size]` — логиты по словарю

### 3. Специальные токены

Для корректной работы seq2seq-модели к словарю GPT-2 (50257 токенов) добавлены два специальных токена:

| Токен | ID | Назначение |
|-------|-----|-----------|
| `EOS` | 50256 | Конец последовательности. Уже есть в GPT-2 (`<\|endoftext\|>`). Сигнализирует декодеру, что генерация завершена. |
| `PAD` | 50257 | Паддинг. Нужен для выравнивания последовательностей разной длины в батче до фиксированного размера. Игнорируется при расчёте loss (`ignore_index`) и через padding-маски в attention. |
| `BOS` | 50258 | Начало последовательности. Подаётся как первый токен на вход декодера, запуская авторегрессионную генерацию. Без него декодеру нечего принять на первом шаге. |

Итоговый размер словаря: **50259**.

## Данные

Датасет состоит из 15 видеоклипов (`pt1`-`pt15`), предварительно обработанных в `.pkl`-файлы. Каждый файл содержит:
- `frames` — список кадров области губ (numpy arrays, 64x96x3, uint8)
- `input_ids` — токены фразы (GPT-2 tokenizer)
- `num_frames` — количество кадров

### Скользящее окно

Модель принимает фиксированные 40 кадров и генерирует до 12 токенов. Если в исходном сэмпле больше 40 кадров, он разбивается на несколько обучающих примеров с перекрытием:

- **Кадры:** окно 40, сдвиг 10
- **Токены:** окно 11 (+ BOS/EOS = 12 для декодера), сдвиг 3

Пример для сэмпла с 70 кадрами и 20 токенами:
```
Окно 1: кадры [0:40],  токены [0:11]
Окно 2: кадры [10:50], токены [3:14]
Окно 3: кадры [20:60], токены [6:17]
```

Для коротких клипов (< 40 кадров) — один сэмпл с паддингом нулями, маска `src_padding_mask` сообщает encoder/decoder-attention игнорировать эти позиции.

### Разбиение на сплиты

| Сплит | Клипы | Назначение |
|-------|-------|------------|
| Train | pt1 - pt11 | Обучение параметров модели |
| Validation | pt12 - pt13 | Контроль переобучения, выбор лучшей модели |

## Обучение

Цикл обучения реализован в `python/train.ipynb`.

### Режим обучения (Train)
- **Teacher forcing**: на вход декодера подаётся `[BOS, t1, ..., tK]`, целевая последовательность — `[t1, ..., tK, EOS]`
- **Loss**: `CrossEntropyLoss` с `ignore_index=PAD_TOKEN_ID` — паддинг-токены не влияют на градиенты
- **Оптимизатор**: AdamW (`lr=1e-4`, `weight_decay=1e-4`)
- **Scheduler**: CosineAnnealingLR (от `1e-4` до `1e-6` за 50 эпох)
- **Gradient clipping**: `max_norm=1.0`
- **Метрика**: token accuracy по не-PAD позициям
- Данные перемешиваются каждую эпоху (`shuffle=True`)

### Режим валидации (Validation)
- Выполняется после каждой эпохи обучения
- Модель переводится в `eval()` mode — отключены Dropout и BatchNorm в режиме обучения
- Градиенты не вычисляются (`@torch.no_grad()`)
- По-прежнему используется teacher forcing для подсчёта loss и accuracy
- Лучшая модель (по минимальному `val_loss`) сохраняется в `best_model.pt`

### Режим тестирования (Test)
- Загружается лучшая модель с наименьшим validation loss
- Вычисляются loss и token accuracy на тестовом наборе (pt14-pt15)
- Дополнительно реализован **greedy decode** — авторегрессионная генерация без teacher forcing:
  1. На вход подаётся только видео и токен `BOS`
  2. На каждом шаге модель предсказывает следующий токен (argmax)
  3. Предсказанный токен добавляется к входу декодера
  4. Генерация останавливается при получении `EOS` или достижении максимальной длины (12)
  5. Результат декодируется обратно в текст через GPT-2 tokenizer
