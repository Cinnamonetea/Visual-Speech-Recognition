# Анализ предобучения визуального энкодера: классификация слов (v1)

**Дата:** 24 апреля 2026

## Конфигурация

| Параметр | Значение |
|---|---|
| Задача | Классификация BPE-токенов по окну из 8 кадров |
| Классов | 100 (топ-100 частых токенов, покрывают 53.5% вхождений) |
| CNN | Stem(3→24) + 2× MBConv (24→48→64) |
| Параметры | 30 990 (CNN: 24 490) |
| Train | 6 092 окна из 19 клипов |
| Val | 760 окон из 2 клипов |
| LR | 1e-3, CosineAnnealing, 50 эпох |

## Результаты

| Метрика | Значение |
|---|---|
| Best val_acc | **0.066** (эпоха 20) |
| Random baseline | 0.010 (1/100) |
| Train acc (эп. 50) | 0.100 |
| Val acc (эп. 50) | 0.054 |

### Динамика обучения

```
Epoch   1 | train_acc=0.042 | val_acc=0.055
Epoch  20 | train_acc=0.079 | val_acc=0.059  *  (best)
Epoch  50 | train_acc=0.100 | val_acc=0.054
```

Модель обучается медленно (train_acc: 0.04→0.10 за 50 эпох) и
переобучается (val_acc падает после эпохи 20).

## Анализ

### val_acc=6.6% vs random=1% — модель учится, но слабо

Модель в 6.6× лучше случайного угадывания — CNN **извлекает**
какую-то визуальную информацию. Но 6.6% на 100 классах — это
очень мало. Модель едва различает самые частые слова.

### Причины слабого результата

**1. Задача слишком сложная для такого маленького CNN.**
24 490 параметров CNN должны различить 100 классов по кадрам
64×96 px. Это ~245 параметров на класс — крайне мало.

**2. Привязка токен→кадры неточная.**
Токены распределяются по кадрам равномерно, но реальная
синхронизация речь→видео не линейная. Модель может получать
кадры с неправильным целевым токеном.

**3. Многие классы визуально неразличимы.**
Токены `"the"`, `"a"`, `"to"`, `"in"` — короткие служебные
слова, которые визуально похожи (быстрое движение губ).
Различить их по 8 кадрам (< 1 сек) сложно даже для человека.

**4. 50 эпох — мало для сходимости.**
train_loss всё ещё падает (4.54→3.94), train_acc растёт.
Модель не достигла насыщения.

## Рекомендации для продолжения

1. **Увеличить число эпох** до 150–200 — train_acc ещё растёт
2. **Уменьшить число классов** до 30–50 — упростить задачу
3. **Увеличить окно** до 12–16 кадров — больше контекста
4. **Увеличить LR** для продолжения — модель застряла на плато
